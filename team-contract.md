# Team Work Contract  
Mercedes–Benz Sales Insights Dashboard (2020–2025)

## 1. Purpose

This contract defines how our team will collaborate to successfully complete the DATA 551 dashboard project using the Mercedes-Benz Global Car Sales dataset (2020–2025).  

We aim to maintain professionalism, fairness, accountability, and high-quality standards throughout the project.

---

## 2. Project Scope Awareness

Our dashboard focuses on:Sales trends across fuel types (Petrol, Diesel, Hybrid, Electric); Performance characteristics (Horsepower, Turbo); Pricing relationships; Model comparisons; Color preferences over time

Given the dataset size (~1.8 million rows), we acknowledge that careful data handling, clear task division, and version control discipline are essential.

---

## 3. Work Distribution

Work will be divided across the following major components:

- Data preprocessing and cleaning  
- Feature engineering (e.g., aggregations, summary metrics)  
- Visualization design (Altair charts)  
- Dash layout and interaction logic  
- Documentation and proposal writing  
- User testing and feedback integration  

Each team member will:

- Take ownership of at least one major component
- Contribute to dashboard integration
- Participate in documentation and review

No single member will carry a disproportionate workload.

---

## 4. Meetings and Communication

We will meet at least once per week. Additional short coordination meetings may be scheduled before deadlines. Agendas will be prepared before meetings. Meeting notes will be documented and shared. Communication will primarily occur through (Slack/WeChat/Discord – choose one). Moreovver, Members are expected to respond within 24 hours.

---

## 5. GitHub Workflow

To maintain code integrity:

- Members must pull latest changes before starting work.
- Small, frequent commits are encouraged.
- Major updates must be submitted via Pull Request (PR).
- At least one teammate must review and approve PRs before merging.
- GitHub Issues will be used to track features and bugs.
- Milestone releases will only be created once all members approve.

---

## 6. Quality Standards

We commit to:

- Writing readable and well-documented code
- Following data visualization best practices
- Designing user-centered dashboard interactions
- Ensuring filters and charts are logically consistent
- Testing interactions before merging into main branch

Our goal is to build a dashboard that resembles a professional business analytics tool.

---

## 7. Accountability

If a member anticipates reduced availability, they must inform the team in advance.

If deadlines are missed without communication:

1. The issue will be discussed internally.
2. Responsibilities may be redistributed if necessary.
3. Persistent lack of contribution may be escalated to the instructor.

---

## 8. Conflict Resolution

We will discuss respectfully during a scheduled meeting, decisions will be made by consensus or majority vote. If unresolved, we will consult the course instructor.
