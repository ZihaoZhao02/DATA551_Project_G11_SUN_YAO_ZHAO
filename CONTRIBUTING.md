# Contributing

Thanks for your interest in contributing!

## Code of Conduct
By participating in this project, you agree to follow our
[Code of Conduct](./CODE_OF_CONDUCT.md).

## How to submit an Issue
1. Go to the **Issues** tab in GitHub.
2. Click **New issue**.
3. Include:
   - A clear title
   - What you observed (problem/request)
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Screenshots or examples (if helpful)

## How to submit a Pull Request (PR)
1. Create a new branch from `main`:
   - `feature/<short-name>` or `fix/<short-name>`
2. Make changes and commit with clear messages.
3. Push your branch and open a PR to `main`.
4. In the PR description, include:
   - What you changed
   - Why you changed it
   - How to test it (if applicable)
5. Request a review from at least one teammate before merging.

## Basic quality checklist
- The app runs without errors
- Code is readable and organized
- No sensitive/private data is included
